## 1. 게임 개요

### 1.1 핵심 컨셉
**"역던전 크롤러 (Reverse Dungeon Crawler)"**
- 플레이어: 여성 워록 (의뢰받은 암살자)
- 목표: 기사 사령관(Knight Commander)을 던전 내에서 암살
- 설정: 왕의 비밀 의뢰를 수행하는 다크 판타지

### 1.2 장르 믹스
| 장르 | 역할 |
|------|------|
| **덱 빌딩** | 카드 수집/편집으로 전략 구성 |
| **타워 디펜스** | 몬스터/트랩 배치로 방어 구축 |
| **오토 배틀러** | 전투는 자동 진행 + 개입 |

## 2. 시스템 분석

### 2.1 게임 플로우
```
INIT → DIALOGUE → PREPARATION → PLACEMENT → COMBAT → RESULT
         ↓              ↑            ↓         ↓
      스토리 진행    덱 편집/정보    배치 페이즈  오토 배틀
```

### 2.2 카드 시스템
| 타입 | 용도 | 현재 카드 |
|------|------|----------|
| **SUMMON** | 방에 유닛 배치 (최대 4마리) | Slime (1), Skeleton (2) |
| **TRAP** | 방에 함정 배치 (방당 1개, 배타적) | Spike Trap (1) |
| **SPELL** | 전투 중 직접 사용 (미구현) | Fireball (3) |

### 2.3 던전 구조
- **선형 구조**: 방이 일렬로 연결 (Darkest Dungeon 스타일)
- **방 개수**: `5 + 현재 레벨`
- **방 타입**: START → NORMAL × n → BOSS

## 3. 밸런스 현황 및 문제점

### 3.1 현재 수치 분석

#### 기사 사령관
| 스탯 | 값 | 비고 |
|------|-----|------|
| HP | 100 | 최종 보스급 |
| ATK | 15 | 매우 높음 |
| DEF | 5 | 방어력 존재 |

#### 플레이어 카드/유닛
| 카드 | 코스트 | HP | ATK | 특수 |
|------|--------|-----|-----|------|
| Slime | 1 | 10 | 2 | - |
| Skeleton | 2 | 15 | 4 | Range 2 (미구현) |
| Spike Trap | 1 | - | - | 5 즉시 피해 |
| Fireball | 3 | - | - | 10 피해 (미구현) |

### 3.2 **치명적 밸런스 문제**

#### 문제 1: 기사의 압도적 화력
```
기사 ATK 15 vs 슬라임 HP 10 = 원킬
기사 ATK 15 vs 스켈레톤 HP 15 = 2턴 킬
```
> **결론**: 소환수가 기사에게 피해를 주기 전에 전멸

#### 문제 2: 방어력 계산의 불균형
```
슬라임 ATK 2 - 기사 DEF 5 = -3 → 최소 1 데미지
스켈레톤 ATK 4 - 기사 DEF 5 = -1 → 최소 1 데미지
```
> **결론**: 아군 유닛이 기사에게 주는 피해가 미미함

#### 문제 3: 킬 필요 유닛 수 계산
```
기사 HP 100 ÷ 슬라임 피해 1 = 100턴 필요
기사 HP 100 ÷ 트랩 피해 5 = 20개 트랩 필요
```
> **결론**: 현재 밸런스로는 클리어 불가능에 가까움

#### 문제 4: 마나 시스템 미구현
- 카드에 코스트가 있지만 마나 자원이 없음
- 무한 배치 가능 → 코스트 의미 없음

#### 문제 5: 레벨 스케일링 부재
- 레벨 증가 시 방 개수만 증가
- 기사 스탯, 보상, 카드풀 변화 없음

## 4. 밸런스 개선 제안

### 4.1 스탯 리밸런싱

#### 방안 A: 기사 너프 + 유닛 버프
| 대상 | 변경 전 | 변경 후 |
|------|--------|--------|
| 기사 HP | 100 | 50~60 (레벨 스케일) |
| 기사 ATK | 15 | 8~10 |
| 기사 DEF | 5 | 2~3 |
| 슬라임 ATK | 2 | 4~5 |
| 스켈레톤 ATK | 4 | 6~8 |

#### 방안 B: 전투 시스템 변경 (추천)
- **동시 공격**: 기사와 모든 아군이 동시에 공격
- **집중 공격 보너스**: 같은 방에 유닛 수 × ATK 보너스
- **범위 공격**: 스켈레톤 Range로 인접 방에서 지원 가능

### 4.2 마나 시스템 설계

```
턴당 마나: 3 (기본) + 레벨 보너스
마나 저장: 다음 턴으로 이월 불가 (매 배치 페이즈 리셋)
```

| 카드 | 현재 코스트 | 제안 코스트 |
|------|------------|------------|
| Slime | 1 | 1 |
| Skeleton | 2 | 2 |
| Spike Trap | 1 | 1 |
| Fireball | 3 | 2 (전투 중 사용) |

### 4.3 레벨 스케일링 공식

```javascript
// 제안: 레벨별 기사 스탯
knightStats = {
    hp: 30 + (level * 20),      // Lv1: 50, Lv5: 130
    atk: 5 + (level * 2),       // Lv1: 7, Lv5: 15
    def: 1 + Math.floor(level / 2)  // Lv1: 1, Lv5: 3
}
```

## 5. 고유한 재미 요소 (Unique Selling Points)

### 5.1 역설적 롤플레이
| 요소 | 설명 |
|------|------|
| **악당 시점** | 일반적인 용사 vs 마왕 구도를 뒤집음 |
| **정의의 적** | 플레이어가 "영웅"을 죽여야 하는 도덕적 딜레마 |
| **고용된 암살자** | 단순 악역이 아닌 회색지대의 캐릭터 |

> **플롯 트위스트 가능성**: 기사 사령관이 사실 선역이고, 왕이 진짜 악역?

### 5.2 정보 기반 전략
```
준비 페이즈에서 적 정보 확인 → 덱 커스터마이징 → 배치 최적화
```
- **강점**: "준비가 곧 실력"이라는 메시지 전달
- **확장 가능성**: 레벨별 다른 기사 타입 (궁수, 마법사, 성기사 등)
- **핵심 원칙**: 덱 편집(카드 제거/교체)은 **무료**여야 함
  - 적 파티가 민첩 위주면 → 트랩 중심 덱
  - 힐러 대동 시 → 고화력 집중 or 힐러 우선 제거 스펠
  - 이런 적응이 자유로워야 전략의 재미가 살아남

### 5.3 배치 타이밍의 긴장감
| 딜레마 | 트레이드오프 |
|--------|-------------|
| 초반 배치 | 확실한 피해 but 자원 소모 |
| 후반 배치 | 기사 HP 확인 후 투자 but 시간 부족 |
| 분산 배치 | 안정적 but 각개격파 당함 |
| 집중 배치 | 강력 but 빈 방 리스크 |

### 5.4 덱 빌딩의 리플레이성
- 런마다 다른 덱 구성 시도 가능
- "트랩 특화", "물량전", "스펠 중심" 등 빌드 다양화

## 6. 추가 시스템 제안

### 6.1 카드 시너지 시스템
```
[조합 예시]
슬라임 + 슬라임 = 킹 슬라임 (HP 30, ATK 8)
스켈레톤 + 트랩 = 리치 (원거리 + 트랩 설치 능력)
```

### 6.2 기사 파티 시스템 (복수 적)
```
레벨 1: 기사 단독
레벨 2: 기사 + 종자(Squire)
레벨 3: 기사 + 궁수 + 힐러
...
```
> 전략 복잡도 증가, 타겟 우선순위 결정 필요

### 6.3 특수 방 타입
| 방 타입 | 효과 |
|--------|------|
| **TREASURE** | 클리어 시 추가 골드/카드 |
| **SHRINE** | 유닛 버프/디버프 |
| **AMBUSH** | 추가 적 등장 |
| **REST** | 기사 HP 회복 (!) |

### 6.4 워록 스킬 트리
```
패시브 선택 (런 시작 시):
├─ 어둠의 힘: 소환수 ATK +2
├─ 함정 전문가: 트랩 피해 2배
├─ 마나 증폭: 턴당 마나 +1
└─ 사령술: 죽은 유닛 50% 확률 부활
```

## 7. 플롯 트위스트 시나리오 제안

### 7.1 "진실은 반전된다"
> **반전**: 기사 사령관은 왕의 폭정에 반대하는 의인이었다

**트리거 조건**: 특정 레벨 도달 or 특정 대화 선택지

**서사 전개**:
1. 초반: 왕의 명령 수행하는 암살자
2. 중반: 기사 사령관의 편지/일지 발견 (던전 내 힌트)
3. 후반: 진실 인지 후 선택지
   - A) 의뢰 완수 → 다크 엔딩
   - B) 왕을 배신 → 트루 엔딩
   - C) 양쪽 모두 제거 → 시크릿 엔딩

### 7.2 게임플레이 연동
| 루트 | 최종 보스 | 난이도 |
|------|----------|--------|
| 다크 엔딩 | 기사 사령관 | 보통 |
| 트루 엔딩 | 왕의 근위대 | 어려움 |
| 시크릿 엔딩 | 양측 연합군 | 극악 |

## 8. 우선순위 정리

### 즉시 필요 (Critical)
1. **밸런스 패치**: 기사/유닛 스탯 조정
2. **마나 시스템 구현**: 카드 코스트 의미 부여
3. **스펠 카드 작동**: 전투 중 개입 가능하게

### 중요 (High)
4. **레벨 스케일링**: 난이도 곡선 구현
5. **카드 획득 시스템**: 상점/보상
6. **Range 속성 구현**: 스켈레톤 원거리 공격

### 향후 과제 (Medium)
7. **크래프팅 시스템**: 카드 합성
8. **기사 파티 시스템**: 복수 적
9. **특수 방 타입**: 던전 다양화

### 후순위 (Low)
10. **플롯 트위스트**: 멀티 엔딩
11. **워록 스킬 트리**: 런 차별화
12. **사운드/이펙트 폴리싱**

## 9. 결론

### 강점
- **독창적 컨셉**: "역던전"이라는 신선한 관점
- **명확한 게임 루프**: 준비 → 배치 → 전투의 깔끔한 흐름
- **확장 가능한 구조**: 카드/적/던전 추가 용이

### 약점
- **밸런스 붕괴**: 현재 수치로는 게임 진행 불가
- **미완성 시스템**: 마나, 스펠, 크래프팅 미구현
- **콘텐츠 부족**: 카드 5종, 적 1종

### 최종 평가
> "뼈대는 탄탄하나, 밸런스와 시스템 완성이 시급함"

코어 루프의 재미 검증을 위해 **밸런스 패치 + 마나 시스템**을 최우선으로 구현할 것을 권장합니다.
